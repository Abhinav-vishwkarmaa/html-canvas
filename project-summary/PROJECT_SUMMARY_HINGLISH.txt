========================================================================
PROJECT SUMMARY - DYNAMIC SUBSCRIPTION FLOW BUILDER (TEMPLATECRAFT)
========================================================================

------------------------------------------------------------------------
YE PROJECT KISLIYE BANA HAI?
------------------------------------------------------------------------
TemplateCraft ek Dynamic Subscription Flow Builder hai. Ye platform marketers aur campaign managers ke liye bana hai jo mobile advertising campaigns chalate hain. Alag-alag countries aur operators (jaise Airtel, Zain, Jio) ke liye unique subscription pages aur funnels (landing pages) banane padte hain.

TemplateCraft unhe ye freedom deta hai ki wo:
1. Ek visual drag-and-drop editor (GrapesJS) ka use karke custom pages (Home, Confirm, OTP, Thank You, Blocked, aur Error pages) design kar sakein.
2. Custom Page-Flow Configurator ke throw visual mapping create kar sakein (kis page ke baad kaun sa page load hoga aur kis validation logic/condition ke basis par).
3. Partner traffic link parameters (`vid`, `aff_id`, `click_id`) capture aur attribute kar sakein taaki advertiser performance track ho sake.
4. Elasticsearch search engine ka use karke high-volume click/event telemetry audit search logs analyze kar sakein.

------------------------------------------------------------------------
PROJECT ME KYA KYA COMPLETE HO GAYA HAI?
------------------------------------------------------------------------
1. Admin Authentication: Email aur password ke sath secure login/register complete hai. JWT token Bearer Authorization header me storage management context ke thrown handle hota hai.
2. Campaigns Management: Campaigns configuration panels, country/operator maps, vendor assignment, aur dynamic flow graphs mappings fully complete hain.
3. Canvas Template Editor: GrapesJS visual builder integration customized blocks aur modules ke sath complete hai.
4. Media Upload Module: S3 Cloud Storage, Cloudinary, aur Local storage providers automatic fallbacks support complete hai.
5. Dynamic Page-Flow Graph Engine: Campaigns page routing flow ko config graphs ke rules ke runtime evaluation par process karta hai (MSISDN_ONLY, OTP_ONLY, BOTH modes support).
6. Click Attribution & Partners Module: Vendors aur Affiliates tracking links check resolve karne ka mechanism taiyar hai.
7. Elasticsearch Logging System: Funnel transactions events backend se direct Elasticsearch index me ingest hote hain, jo quick logs retrieval aur query filter dashboard facilitate karta hai.
8. OTP Verification Subsystem: Custom rate limits, resend delay locks, twilio/msg91/partner providers adapters, aur UI timer components fully wired aur verified hain.
9. Admin Profile, Settings & Dynamic Logs: Admin apni details update kar sakta hai, password change kar sakta hai aur local Timezone customize kar sakta hai. `CampaignLogsPage.jsx` ke logs select kiye timezone ke basis par format hote hain, aur `CampaignDetailPage.jsx` par header aur quick actions mein direct campaign logs redirection button link kiya gaya hai jo automatically us campaign ke logs filter karke show karta hai. Saath hi, logs list par click karne se **User Session Flow Timeline Modal** khulta hai jo visitor journey show karta hai, aur table columns (Event Name, Vendor, Affiliate, Click ID, aur MSISDN) par click karke direct interactive filtering active ho jati hai.
10. Production Deployment & PM2 Setup: `deploy.sh` script build aur setup automate karti hai. PM2 `ecosystem.config.cjs` aur backend/frontend `start.sh` wrapper scripts use hote hain taaki deploy se pehle dynamic builds check ho sakein.
11. Secure Docker Services: Elasticsearch public port ko Docker compose file me localhost `127.0.0.1` par bind karke secure kiya gaya hai taaki ransomware risks protect ho sakein.
12. High-Traffic Scaling & Database Optimizations: Campaign routes, API configurations aur partner attribution checks ko resolve karne ke liye 15-second TTL cache (`SimpleCache`) implement kiya gaya hai (Phase 1). Telemetry writes overhead aur database read-locks minimize karne ke liye local event queue buffer (`eventBuffer` with 5s background flush / 100 capacity) aur 10s visit lookup caching implement kiya gaya hai (Phase 3).
13. Interactive Hotspots aur Client-Side Routing: GrapesJS editor me image hotspots ab direct `SUBSCRIBE` action ke sath map ho sakte hain (`data-action="SUBSCRIBE"`). Isse custom graphic elements/banners par transparent subscription button overlay lagana easy ho gaya hai. Sath hi, Shadow DOM rendering ke andar internal page links (jaise HOME, OTP, CONFIRM, etc.) ko client-side select karke search query parameter (`step`) par load kiya jata hai bina dynamic page reloads ke, jisse frontend response user navigation smooth rehta hai.

BACKEND ME KYA HUA HAI?
------------------------------------------------------------------------
Backend ko NestJS framework ka use karke banaya gaya hai. Data access TypeORM ORM ke dynamic adapters handle karte hain. Elasticsearch indexing `@elastic/elasticsearch` package ke throw client connection verify karta hai. APIs response envelope standardization global Interceptors compile karte hain. Ab Profile profile updates aur Password change (`change-password.dto.ts`) routes add kiye gaye hain. In-memory caching layers (SimpleCache) aur telemetry batch event buffer queue backend processing optimization complete karte hain.

------------------------------------------------------------------------
FRONTEND ME KYA HUA HAI?
------------------------------------------------------------------------
Frontend React 19 aur Vite compiler par based hai. Layout styling Tailwind CSS v4 use karti hai. Zustand global states and routing context manage karta hai.
Naye screens like `VendorsPage.jsx` (vendors/affiliates mapping), `FlowBuilderPage.jsx` (interactive flow chart logic designer), `CampaignLogsPage.jsx` (ES query builder and logs viewer with user selected timezone formatting) aur `ProfilePage.jsx` (admin profile aur credentials setup manager) add ho gaye hain.

------------------------------------------------------------------------
DATABASE ME KYA HUA HAI?
------------------------------------------------------------------------
Database structure TypeORM auto-migrations ke throw database schema load karta hai:
- users: Admin metadata.
- templates: JSON GrapesJS layouts.
- campaigns: Details, including vendor_id, verification_mode, aur flow_config graph configurations.
- campaign_pages: Slots linking pages.
- vendors / affiliates: Tracking codes mapping.
- api_configs: Outbound verify details.
- visits / visit_events: Attribution details columns (`vendor_id`, `affiliate_id`, `click_id`, etc.) ke sath updated click logs.
- otp_requests: Hashed codes parameters.

MAJOR MODULES DETAILED EXPLANATION (2-3 Paragraphs Each)

Campaign module campaigns metadata store karne ke sath custom flows layout build karne ki functionality deta hai. Backend Search Service events data format coordinate karti hai, aur frontend page search parameter filters (dates ranges, click IDs, operators tags) parse karke sub-second logging search queries screen load karta hai.
